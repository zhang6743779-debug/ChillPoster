围绕日常维护组织功能

这里按实际菜单和模块解释 ChillPoster 做什么，不写虚的功能名，也不把未确认的页面效果画成成品。

01

Emby 封面系统

手动封面、封面设计、自动封面、备份恢复、字体管理和模板管理，统一维护媒体库视觉风格。

02

115 网盘一条龙

资源转存、目录拓扑、媒体整理、失败隔离、定时清理、秒传上传和 Life 事件监控协同工作。

03

302 网关

根据配置启动一个或多个网关服务，支持多 Emby、多端口映射和 115 资源访问链路。

04

媒体整理与 STRM

识别测试、TMDB 信息、重命名模板、二级分类、整理记录和 STRM 同步让入库过程更稳定。

05

发现推荐与缺集治理

发现内容、统计缺集、比对 TMDB、跳转资源搜索或 MoviePilot 订阅，形成补齐闭环。

06

通知与运维

微信、Telegram、Webhook、Docker 管理、系统健康、Emby 任务中心与版本升级集中在一个后台。

一条龙链路怎么跑

下面是 ChillPoster 里真实存在的协作关系：发现、转存、整理、STRM、刷新和通知。每一步都对应后台配置或任务记录。

资源发现 通过发现推荐、RSS、缺集统计或 MoviePilot 联动找到目标内容。
115 转存 把资源统一转入标准拓扑目录，为自动整理和 STRM 同步做准备。
自动整理 扫描、识别、刮削、重命名、二级分类，并记录整理结果。
STRM 与刷新 生成或更新 STRM 文件，触发 Emby 媒体库刷新和计划任务。
通知闭环 整理成功、失败、入库聚合、签到和系统任务都能推送到通知渠道。

几分钟部署到 NAS，长期运行更省心

正式使用推荐 Docker 或 Docker Compose。本地开发可以用 Python + Vite 构建前端后启动。

Docker Compose 示例

services: chillposter: image: chillne/chillposter:latest container_name: chillposter restart: unless-stopped ports: - "5256:5256" - "8011:8011" volumes: - ./config:/app/config - ./fonts:/app/fonts - ./templates:/app/templates - ./layouts:/app/layouts - ./backups:/app/backups

管理端口

默认访问 http://NAS_IP:5256。不建议直接暴露公网，远程访问优先使用 VPN 或带鉴权的反代。

网关端口

如果启用 302 网关，需要额外映射配置中的网关端口，例如 8011。
