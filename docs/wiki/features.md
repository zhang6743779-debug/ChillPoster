# 功能指南

本页按产品模块介绍 ChillPoster 的主要功能与典型使用方式。

## 仪表盘

仪表盘聚合系统状态、运行任务、快捷入口与核心指标。它适合作为日常入口：

- 查看当前后台任务
- 跳转媒体整理、STRM、RSS、封面任务
- 观察系统健康与关键服务状态
- 快速进入常用配置

## 封面系统

封面系统包含：

- 手动封面
- 封面设计
- 自动封面
- 封面备份
- 字体管理
- 模板管理
- 翻译配置

推荐工作流：

1. 上传或选择字体。
2. 创建封面模板。
3. 对单个媒体测试预览。
4. 批量生成封面。
5. 应用前创建备份套件。
6. 如效果不符合预期，从备份恢复。

## 资源转存

资源转存用于把外部资源转入 115 指定目录，并为后续整理、STRM 与 Emby 入库做准备。

适合场景：

- 手动提交资源链接
- 配合发现推荐或订阅结果
- 统一进入一条龙转存目录
- 转存后由 Life 监控触发自动整理

## 媒体整理

媒体整理是 ChillPoster 的核心自动化链路。它负责：

- 扫描源目录
- 识别电影与剧集
- 匹配 TMDB 元数据
- 按规则重命名
- 按二级分类移动或整理
- 刷新 Emby 媒体库
- 输出整理记录与通知

整理前建议：

- 明确源目录和目标目录，避免把媒体库目录当作源目录。
- 使用识别测试验证文件名。
- 先小范围手动整理，再开启监控。
- 为失败目录保留足够空间，方便排障。

## 二级分类

二级分类用于把媒体按电影、电视剧、动漫、纪录片等规则落到不同媒体库或目录。

可维护内容：

- 分类规则
- 子分类设置
- Emby 同步配置
- 默认规则恢复

修改分类后，请注意旧分类路径不会自动从 Emby 中删除，需要在 Emby 后台手动清理不再使用的媒体库或路径。

## 整理记录

整理记录用于查看历史入库结果、失败原因和任务上下文。排障时优先查看：

- 原文件名
- 识别结果
- 目标路径
- 失败信息
- 对应任务 run id

## STRM 同步

STRM 同步用于生成或更新 `.strm` 文件。常用于远程网盘资源映射到本地媒体库。

典型策略：

- 每个媒体库一个同步任务
- 每日低峰时段全量同步
- 资源转存或整理后按需增量同步
- 同步完成后刷新 Emby

## 发现推荐

发现推荐用于聚合外部内容源和影视参考信息，帮助你找到值得订阅或转存的内容。它可以和 MoviePilot、资源搜索和缺集统计形成入口闭环。

## 缺集统计

缺集统计用于定位本地媒体库与 TMDB 剧集信息之间的差异。

适合场景：

- 查看本地缺少哪些季集
- 比对 TMDB 集数
- 打开 Emby 对应条目
- 发起 MoviePilot 订阅
- 进入资源搜索补齐

## RSS 真实库

RSS 真实库适合将订阅源转换成定时任务，并把资源落到指定目录。

能力包括：

- 创建任务
- Cron 定时
- 手动运行
- 启停任务
- 删除任务
- 与标准路径绑定

## 独立真实库

独立真实库用于和主一条龙链路相对解耦的真实库同步场景。适合更谨慎的资源处理策略，或需要单独观察的一组任务。

## Emby 任务中心

Emby 任务中心可读取并管理 Emby 计划任务：

- 查看任务分组
- 手动运行任务
- 停止任务
- 配置触发器
- 开启任务通知
- 查看任务进度

## Docker 管理

Docker 管理用于容器级运维。适合在 NAS 上检查镜像、容器状态、升级与计划重启。

## Forward 爱影模块

Forward 模块用于爱影资源查询、115 分享预览、转存与直链播放。根据爱影 Token、TG ID 和外部服务可用性，可能需要单独验证连接。

## Webhook

Webhook 用于接收外部系统事件。ChillPoster 内置持久化队列 worker，降低瞬时事件丢失风险。适合接入 Emby 删除同步、外部订阅系统或自定义自动化。
