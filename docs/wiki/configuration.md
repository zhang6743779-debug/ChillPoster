# 核心配置

ChillPoster 的配置大多保存在 `config/` 目录。该目录包含账号、Token、运行状态、任务记录和缓存，默认不应提交到 Git。

## 首次配置顺序

推荐按下面顺序完成配置：

1. `账户管理`：修改默认登录信息。
2. `Emby 配置`：填写 Emby 地址与 API Key。
3. `115 配置`：绑定 115 账号、根目录与转存目录。
4. `302 配置`：设置网关端口、Emby 访问模式和 115 访问方式。
5. `媒体整理`：配置源目录、目标媒体库、分类规则与识别策略。
6. `STRM 同步`：创建同步任务。
7. `通知配置`：配置微信、Telegram 或 Webhook。
8. `TMDB 配置`：配置元数据源与代理。

## Emby 配置

Emby 是 ChillPoster 的主要媒体库目标。至少需要：

- Emby Server 地址
- API Key
- 可选 public host，用于外部访问链接

相关能力：

- 媒体库读取
- 封面应用与备份
- 媒体库刷新
- Emby 任务中心
- 缺集统计
- 302 网关访问

## 115 配置

115 配置用于网盘目录浏览、转存、整理、STRM、秒传、上传、清理与 Life 事件监控。

建议建立标准拓扑：

```text
影视库/
├── 转存目录/
├── 整理目录/
├── 媒体库/
└── 失败目录/
```

实际目录名称可在 UI 中配置。标准拓扑的意义是让转存源、整理目标、失败隔离和媒体库同步关系保持清晰。

## 302 配置

302 配置负责把 Emby 播放请求导向 115 资源访问链路。常见配置包括：

- 115 账号列表
- Emby 服务器列表
- 网关端口
- 网关访问模式
- 资源转存目录
- 一条龙标准目录绑定

如果配置了多个 Emby 且启用了多个网关端口，主程序会按配置启动多个网关服务。

## 媒体整理配置

媒体整理模块用于把源目录中的媒体文件整理到目标媒体库目录，并生成可被 Emby/Jellyfin 识别的结构。

关键配置：

- 整理源目录
- 目标媒体库目录
- 失败目录
- 115 账号索引
- 命名模板
- 二级分类规则
- Life 事件监控
- 额外监控目录
- TMDB 与 ffprobe 识别策略

建议先使用 `识别测试` 验证规则，再开启自动整理。

## STRM 同步

STRM 同步适合网盘资源场景。它可以把远端资源映射为 `.strm` 文件，让 Emby 以媒体库形式管理。

建议：

- 按媒体库或资源目录拆分任务。
- 先手动执行一次全量同步。
- 确认 Emby 刷新结果后再开启定时同步。

## 通知配置

通知用于让后台任务可观察。推荐至少配置一种通知渠道：

- 微信通知
- Telegram Bot
- Telegram 账号监听
- Webhook

适合通知的事件：

- 媒体整理完成或失败
- 剧集入库聚合结果
- 签到结果
- Webhook 事件
- 系统任务异常

## MoviePilot 配置

MoviePilot 配置用于和外部 MoviePilot 服务联动，例如缺集补齐、订阅和资源搜索入口。填写前请确认 MoviePilot 服务地址与认证信息可用。

## 代理与 TMDB

如果服务器访问 TMDB 不稳定，可在代理配置中设置代理地址。主程序启动时会尝试读取全局配置并应用到 TMDB 客户端。

## 配置文件安全

不要提交以下文件：

- `config/*.json`
- `config/*.log*`
- `config/*.db`
- 包含 Cookie、Token、API Key、账号密码的任何文件

仓库只跟踪默认二级分类规则：

```text
config/media_organize_category_rules.json
```
