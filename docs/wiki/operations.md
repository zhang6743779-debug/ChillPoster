# 运维手册

本页面向长期运行 ChillPoster 的用户，覆盖健康检查、日志、备份、排障与升级前检查。

## 日常检查

建议定期检查：

- UI 是否能打开
- `/api/version` 是否返回预期版本
- 系统健康页面是否有异常
- 后台任务是否存在长时间 running
- 302 网关端口是否可访问
- Emby 媒体库刷新是否正常
- 115 账号是否仍可用
- 通知渠道是否能收到消息

## 日志位置

常见日志位于：

```text
config/app.log
config/app.log.1
config/app.log.2
config/app.log.3
```

排障时优先按时间定位：

1. UI 操作时间
2. 后台任务 run id
3. 对应模块关键词，例如 `MediaOrganize`、`Gateway`、`RSS`、`STRM`、`HDHive`

## 任务状态

任务状态与进度会持久化到配置目录。服务重启后，旧 running 任务会被标记为中断，部分整理任务支持重新扫描续跑。

常见任务类型：

- 媒体整理
- STRM 同步
- RSS
- 封面备份与生成
- 升级任务
- 系统任务

## 备份策略

升级、迁移或重装前建议备份：

```text
config/
fonts/
templates/
layouts/
backups/
```

其中 `config/` 最重要，包含账号、任务、运行状态和缓存。`templates/`、`fonts/`、`layouts/` 决定封面系统效果。`backups/` 保存封面备份套件。

## 常见问题

### UI 能打开但网关不可用

检查：

- 302 配置中的网关端口是否启用。
- Docker 是否映射了对应端口。
- Emby 配置是否有正确地址和 API Key。
- 115 配置是否仍然有效。
- 网关日志中是否有连接池或认证错误。

### 媒体整理没有自动触发

检查：

- 媒体整理中是否开启 Life 事件监控。
- 源目录和目标目录是否配置完整。
- 源目录是否被安全检查判定为疑似媒体库目录。
- 115 客户端是否可用。
- 额外监控目录是否启用。

### 媒体识别不准确

建议：

- 使用识别测试单独验证名称。
- 调整重命名模板或分类规则。
- 确认 TMDB 配置和代理可用。
- 对特殊文件名先进行人工规范化。
- 检查是否开启了仅文件名模式或 ffprobe 策略。

### Emby 没有看到新内容

检查：

- 目标目录是否在 Emby 媒体库路径中。
- 整理后是否触发 Emby 刷新。
- STRM 文件是否生成到 Emby 可扫描路径。
- Emby 任务中心中媒体库扫描任务是否成功。
- Emby 容器是否能访问对应挂载路径。

### 通知没有收到

检查：

- 通知配置是否保存。
- 微信、Telegram 或 Webhook 凭据是否有效。
- 网络代理是否影响通知接口。
- 任务本身是否产生了通知事件。
- 日志中是否有发送异常。

## 性能建议

- 大规模媒体库首次扫描时，尽量在低峰时段运行。
- 大量 STRM 同步任务建议分批或错峰。
- Emby 刷新不宜过于频繁，可通过任务中心观察执行耗时。
- 发现页和媒体库缓存会在启动时预热，首次启动稍慢属于正常现象。

## 安全建议

- 不要公开暴露管理端口 `5256` 到公网。
- 如需远程访问，优先使用 VPN、内网穿透认证或反向代理鉴权。
- 定期更换 Emby API Key、Webhook Token、Telegram Bot Token 等敏感凭据。
- 不要把 `config/` 目录上传到公开仓库。

## 迁移步骤

1. 停止旧容器。
2. 备份 `config/`、`fonts/`、`templates/`、`layouts/`、`backups/`。
3. 在新机器创建相同挂载目录。
4. 复制备份文件。
5. 使用相同镜像或目标版本启动容器。
6. 登录 UI，检查配置、任务、网关端口和通知。
7. 手动运行一个低风险任务验证链路。
